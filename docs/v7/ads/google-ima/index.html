Google IMA

react-native-video includes built-in support for Google IMA SDK (Interactive Media Ads) on Android and iOS. This enables you to monetize your video content with AVOD (Ad-Supported Video on Demand).

Installation‚Äã

To enable IMA SDK support, refer to the installation section.

Usage‚Äã

To display ads, pass the adTagUrl prop to the Video component. The adTagUrl should be a VAST-compliant URI.

import Video from 'react-native-video';

function App() {
  return (
    <Video
      source={{ uri: 'https://example.com/video.mp4' }}
      adTagUrl="https://pubads.g.doubleclick.net/gampad/ads?iu=/21775744923/external/vmap_ad_samples&sz=640x480&cust_params=sample_ar%3Dpremidpostoptimizedpodbumper&ciu_szs=300x250&gdfp_req=1&ad_rule=1&output=vmap&unviewed_position_start=1&env=vp&impl=s&cmsid=496&vid=short_onecue&correlator="
      style={{ width: '100%', height: 300 }}
    />
  );
}

Events‚Äã

To receive events from the IMA SDK, use the onReceiveAdEvent prop:

<Video
  source={{ uri: 'https://example.com/video.mp4' }}
  adTagUrl="your-ad-tag-url"
  onReceiveAdEvent={(event) => {
    console.log('Ad event:', event);
    
    switch (event.type) {
      case 'STARTED':
        // Ad started playing
        break;
      case 'COMPLETED':
        // Ad completed
        break;
      case 'ALL_ADS_COMPLETED':
        // All ads finished
        break;
    }
  }}
/>

The full list of supported events is available in the AdEvent types.

Localization‚Äã

To change the language of the IMA SDK, use the adLanguage prop:

<Video
  source={{ uri: 'https://example.com/video.mp4' }}
  adTagUrl="your-ad-tag-url"
  adLanguage="fr" // French
/>

The list of supported language codes is available in the Google IMA documentation.

note
  • iOS uses the system language by default
  • Android defaults to en (English)

Picture-in-Picture Considerations‚Äã

Video ads cannot start when Picture-in-Picture (PiP) mode is active on iOS. See the Google IMA SDK documentation for more details.

Recommended approach: If you're using custom controls, hide the PiP button when receiving the STARTED event and show it again when receiving the ALL_ADS_COMPLETED event.
