Google DAI

This feature is part of the Pro Player plan. Pricing information can be found here.

Google DAI (Dynamic Ad Insertion) provides server-side ad insertion, delivering a seamless viewing experience with no buffering between content and ads.

Why Google DAI?‚Äã

FeatureGoogle IMA (Client-side)Google DAI (Server-side)
Ad insertionClient-sideServer-side
BufferingMay have bufferingSeamless, no buffering
Ad blockersCan be blockedHarder to block
Stream qualitySeparate streamsSingle unified stream
ImplementationSimplerMore complex

Features‚Äã

  • Seamless playback - No buffering between content and ads
  • Ad blocker resistant - Server-side insertion is harder to block
  • Unified stream - Single stream with ads stitched in
  • Live & VOD support - Works with both live streams and video on demand
  • Analytics integration - Full tracking and reporting capabilities

Basic Usage‚Äã

import Video from 'react-native-video';

function App() {
  return (
    <Video
      source={{
        uri: 'your-dai-stream-url',
        type: 'dai',
        assetKey: 'your-asset-key', // For live streams
        // OR
        contentSourceId: 'your-content-source-id', // For VOD
        videoId: 'your-video-id',
      }}
      onDaiAdEvent={(event) => {
        console.log('DAI event:', event);
      }}
      style={{ width: '100%', height: 300 }}
    />
  );
}

Events‚Äã

DAI provides specific events for ad tracking:

onDaiAdEvent={(event) => {
  switch (event.type) {
    case 'AD_BREAK_STARTED':
      // Ad break beginning
      break;
    case 'AD_BREAK_ENDED':
      // Ad break finished
      break;
    case 'AD_PROGRESS':
      // Ad playback progress
      break;
  }
}}
